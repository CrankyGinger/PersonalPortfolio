Layout With Flexbox

When making websites with CSS, there are different approaches to the task of arranging the content. But, arguably the best way to approach this challenge is flexbox - a way of laying out different elements on a website.

Per the assignment, we were required to achieve a certain layout on a webpage by using flexbox only. More specifically, we had to arrange several smaller elements all within a larger element that would act as a container, the way this would be done would be by putting differently sized containers in rows - for example, one row might have one container, whilst the next would have two, and then the next would have three, so on and so forth.

When arranging the content, there were a fair share of challenges, as it was my first time ever using flexbox. Flexbox offers users a lot of different ways to arrange content, depending on what you’re going for on your site. One of the biggest challenges was figuring out which property to use, as using the wrong one would make it a nightmare to achieve the goal.

When I was facing a struggle with making elements fit within what was essentially a row, I would employ some of CART’s habits of mind - communicate, persist. When I was aiming to arrange elements within a row, it wouldn’t come to me very easily, so I would communicate with my classmates, and ask them questions. After talking to my comrades about the issue at hand, I would persist through the struggle and just keep on trying different methods. Eventually, I completed the assignment and felt great satisfaction after my persistence through the challenge of learning more about CSS.
